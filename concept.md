# Petit Monde Vivant

## Vision du projet

### Résumé

Petit Monde Vivant est un jeu web solo médiéval à très faible APM.

Le joueur n'incarne pas un héros destiné à sauver le monde.

Il est simplement un habitant d'un monde autonome qui continue d'évoluer même lorsqu'il ne le regarde pas.

L'objectif principal n'est pas la victoire.

L'objectif principal est d'observer l'émergence d'histoires, de familles, de villages, de métiers et d'événements au sein d'une société médiévale simulée.

Le jeu doit procurer une sensation de calme, de curiosité et d'attachement au monde.

---

# Les piliers du projet

## Pilier 1 : Le monde est le personnage principal

Le joueur est important.

Mais le véritable protagoniste est le monde lui-même.

Le joueur doit régulièrement se connecter et découvrir que des choses se sont produites sans lui.

Exemples :

* un marchand a fait fortune
* une famille a disparu
* un village s'est enrichi
* une nouvelle route commerciale est apparue
* une forêt a été surexploitée

Le monde doit sembler vivant et autonome.

---

## Pilier 2 : Faible APM

Le jeu ne cherche pas à tester les réflexes.

Le joueur doit pouvoir jouer d'une seule main.

Le gameplay doit rester confortable même après plusieurs heures.

Le plaisir doit provenir de la prise de décision et de l'observation.

Jamais de spam de clics.

Jamais de combat nécessitant de l'exécution rapide.

Jamais de stress artificiel.

---

## Pilier 3 : Progression permanente

Chaque session doit produire un résultat visible.

Même une session de cinq minutes doit permettre au joueur d'avancer.

Le joueur doit avoir la sensation que son histoire progresse continuellement.

---

## Pilier 4 : Histoires émergentes

Les événements les plus intéressants ne doivent pas être écrits à l'avance.

Ils doivent émerger des interactions entre les systèmes.

Objectif :

Le joueur doit pouvoir raconter l'histoire de son monde.

Exemple :

"Le vieux forgeron est mort pendant l'hiver.
Sa fille a repris l'atelier.
Elle est devenue la plus riche du village."

Cette histoire ne doit pas être scriptée.

Elle doit être produite par la simulation.

---

# Ce que le jeu n'est PAS

Le jeu n'est pas un MMORPG.

Le jeu n'est pas un RPG d'action.

Le jeu n'est pas un city-builder.

Le jeu n'est pas un idle game classique basé uniquement sur des nombres qui augmentent.

Le jeu n'est pas un jeu compétitif.

Le jeu n'est pas un jeu de survie.

---

# Boucle principale

Observer le monde.

Choisir une activité.

Produire des ressources.

Influencer légèrement le monde.

Observer les conséquences.

Découvrir les changements.

Recommencer.

---

# Ambiance

Univers médiéval réaliste.

Pas de magie.

Pas de dragons.

Pas de prophétie.

Pas de héros élus.

Inspirations :

* la campagne européenne médiévale
* les villages de WoW Classic
* le sentiment de monde vivant d'Albion
* les histoires émergentes de RimWorld
* les générations de Crusader Kings
* la simulation de Dwarf Fortress

Le ton général doit rester chaleureux et contemplatif.

---

# Philosophie technique

Le gameplay prime toujours sur les graphismes.

Un système simple mais vivant vaut mieux qu'un système complexe invisible.

Chaque nouvelle fonctionnalité doit répondre à la question :

"Le joueur peut-il voir son effet dans le monde ?"

Si la réponse est non, la fonctionnalité doit être reportée.

---

# Ordre de développement

Phase 1 :
Le joueur produit des ressources.

Phase 2 :
Des PNJ produisent des ressources.

Phase 3 :
Les PNJ échangent des ressources.

Phase 4 :
Les PNJ développent des relations.

Phase 5 :
Les familles apparaissent.

Phase 6 :
Les villages évoluent.

Phase 7 :
Le monde raconte sa propre histoire.

Aucune phase ne doit être commencée avant que la précédente soit satisfaisante.

---

# Règle absolue

À tout moment, un joueur doit pouvoir répondre à cette question :

"Qu'est-ce qui a changé dans le monde depuis ma dernière connexion ?"

Si la réponse est "rien", alors le jeu a échoué.
